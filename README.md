# Toytype

Toytype은 웹페이지와 Google Docs 원고에서 오탈자 후보를 찾는 Chrome 확장 프로그램입니다.
내장 `rules.json`으로 결정론 검사를 수행하고, Google Docs에서는 문서 위치 선택, 항목별 적용, AI 보조 검토, AI 발문 삽입, AI 문장 길이 조절, 목차 추출을 제공합니다.

- 현재 버전: `1.0.0`
- 내장 규칙: 8개 카테고리, 6,866개 규칙
- 기본 검사만으로는 문서를 수정하지 않습니다.
- Google Docs에서 사용자가 항목별 **적용** 버튼을 누른 경우에만 해당 선택 영역 교체를 시도합니다.

## 설치

1. Chrome 주소창에서 `chrome://extensions`로 이동합니다.
2. 우측 상단 **개발자 모드**를 켭니다.
3. [압축해제된 확장 프로그램을 로드]를 누르고 이 폴더(`Toytype`)를 선택합니다.
4. 툴바 확장 아이콘에서 Toytype을 확인합니다.

로컬 HTML 파일(`file://`)도 검사하려면 확장 세부정보에서 **파일 URL에 대한 액세스 허용**을 켜세요.

## 기본 사용

### 일반 웹페이지

- 발견된 위치에 물결 밑줄과 배경색을 표시합니다.
- 툴바 배지는 현재 탭의 발견 건수를 보여줍니다.
- 팝업에서 항목을 클릭하면 교정어가 클립보드에 복사됩니다.
- 팝업에서 카테고리별 검사 범위와 사이트별 비활성화를 설정할 수 있습니다.

색상 계열:

| 색 | 카테고리 |
|---|---|
| 빨강 | 표기 변환, 맞춤법, 맨마지막 |
| 보라 | 존대와 복수, 높임말 서술어 |
| 주황 | 공백 1, 공백 2, 공백 3 |

### Google Docs

Google Docs에서는 본문에 밑줄을 긋지 않고, 우하단 패널 목록으로만 표시합니다.

- 버블 클릭: Toytype 패널 열기
- 항목 클릭: 문서 위치 선택 + 교정어 복사
- **적용**: 선택 영역을 교정어로 교체 시도
- **다시 검사**: 현재 문서 모델 기준으로 재검사
- **JSON 업로드**: `rules.json` 구조의 파일을 임시 규칙으로 적용
- **종류별/문서 순서 보기**: 카테고리 묶음 또는 문서 등장 순서로 전환
- **설정**: AI 브리지와 부가 설정 열기
- **추가기능**: AI 교정 생성, AI 발문 삽입, AI 문장 길이 조절, 이미지 추출, 목차 추출
- 좌하단 브리지 배지: 브리지 상태 표시. 꺼져 있으면 클릭해서 재시작 명령을 복사합니다.

내부 Google Docs 문서 모델 접근이 막히면 텍스트 export로 fallback합니다.

## AI 교정 생성

AI 교정은 선택 기능입니다. Chrome 확장은 로컬 CLI를 직접 실행할 수 없으므로, 프로젝트 루트에서 로컬 브리지를 먼저 실행해야 합니다.

```bash
node tools/toytype_ai_bridge.mjs --port 17644
```

위 명령은 foreground 서버라서 실행 후 터미널을 계속 잡고 있습니다. AI 기능을 쓰는 동안 그 터미널을 열어 두고, 중지하려면 [Ctrl-C]를 누릅니다.

브리지를 재시작하려면 다른 터미널에서 다음 명령을 실행합니다.

```bash
node tools/toytype_ai_bridge_ctl.mjs restart --port 17644
```

이 재시작 명령은 백그라운드로 브리지를 띄우고 바로 종료됩니다. 로그는 다음 명령으로 확인합니다.

```bash
tail -f /tmp/toytype-ai-bridge-17644.log
```

재시작 후에도 터미널을 서버로 잡아 두고 싶으면 다음처럼 실행합니다.

```bash
node tools/toytype_ai_bridge_ctl.mjs restart --port 17644 --foreground
```

브리지가 이미 실행 중이면 같은 명령을 한 번 더 실행할 필요가 없습니다. `EADDRINUSE` 또는 “already running” 메시지가 나오면 기존 브리지가 포트를 사용 중이라는 뜻입니다.

설정 페이지에서 지정할 수 있는 항목:

- 브리지 URL: 기본 `http://127.0.0.1:17644`
- AI 엔진: Codex 또는 Claude Code
- Codex/Claude 명령 경로
- 작업 폴더
- 생성 JSON 저장 폴더
- 오래된 생성 JSON 제거 기준
- 요청 제한 시간과 문서 최대 전달 길이
- 목차 추출 최대 깊이
- 항목 선택 시 자동 복사 여부

AI 교정 생성 방식:

1. Google Docs 본문 텍스트를 읽습니다.
2. 별도 fact-check 패스를 먼저 실행합니다. Codex 사용 시 `gpt-5.5`, Claude Code 사용 시 Sonnet을 사용하고, Claude 경로에서는 `WebFetch`/`WebSearch` 도구를 허용합니다.
3. 내장 `rules.json`과 fact-check 보고서를 참고 자료로 함께 전달합니다.
4. AI는 단순 맞춤법보다 비문, 윤문, 내용 오류, 내용 중복, 문서별 용어 일관성을 우선 검토합니다.
5. 결과를 Toytype JSON으로 저장하고, 현재 문서 검사 규칙으로 자동 적용합니다.
6. 같은 문서 id의 generated JSON은 규칙 드롭다운에 표시됩니다.

생성 파일명:

```text
<google-docs-id>-<timestamp>.json
```

패널 드롭다운에는 문서 id 대신 `ai 검토-<timestamp>.json`으로 표시됩니다.

AI 작업 중에는 패널 푸터에 진행 상태와 경과 시간이 고정 표시됩니다. 완료/실패 상태도 사라지지 않고 마지막 결과로 남습니다.

## AI 발문 삽입

Google Docs 패널의 [추가기능] ➝ [AI 발문 삽입]은 현재 커서 위치를 기준으로 원고 문맥을 읽어 200자 안팎의 발문을 생성하고, 즉시 문서에 넣지 않고 문장 제안 JSON에 저장합니다.

- 전체 원고 대신 커서 앞 3,500자와 뒤 2,500자만 로컬 브리지에 전달합니다.
- Codex 사용 시 발문/문장 길이 조절 작업에는 계정에서 허용되는 기본 모델을 쓰되 `model_reasoning_effort="low"`로 낮춰 실행합니다.
- Claude Code 사용 시 발문 작업에는 `fable` 모델 옵션을 붙입니다.
- Google Docs 내부 문서 모델과 커서 위치를 읽을 수 있을 때만 제안을 생성합니다.
- 저장 파일명은 `<google-docs-id>-문장제안.json`이며, 같은 문서에서 반복 실행하면 같은 파일에 누적됩니다.
- 패널 푸터의 [문장 제안] 아이콘에서 전체 제안을 확인할 수 있습니다.
- 제안을 클릭하면 원문이나 이미 반영된 문장 위치를 찾고, 제안문을 클립보드에 복사합니다.

## AI 문장 길이 조절

Google Docs에서 문장을 드래그한 뒤 [추가기능] ➝ [AI 문장 길이 조절]을 누르면 원하는 길이 비율을 입력하는 팝업이 뜹니다.

- 입력 범위는 10%부터 500%까지입니다.
- 선택 원문, 선택 앞뒤 문맥, 목표 비율을 로컬 브리지에 전달합니다.
- AI 응답은 Toytype JSON 규칙으로 저장됩니다.
- 저장 파일명은 발문과 같은 `<google-docs-id>-문장제안.json`이며, 실행할 때마다 규칙이 누적됩니다.

## 목차 추출

Google Docs 패널의 [추가기능] ➝ [목차 추출]은 AI를 사용하지 않습니다.

- Google Docs markdown export와 문서 개요 DOM을 우선 사용합니다.
- H1부터 H5까지 추출합니다.
- 결과는 자동으로 클립보드에 복사됩니다.
- 출력은 `#`, 제목, “추출:” 같은 부가 문구 없이 목차 줄만 생성합니다.
- H2부터는 `_`, H3는 `__`처럼 레벨에 맞춰 언더바를 붙입니다.

## 이미지 추출

Google Docs 패널의 [추가기능] ➝ [이미지 추출]은 현재 문서를 DOCX로 export한 뒤 문서 안의 원본 이미지 파일을 순서대로 ZIP으로 저장합니다.

- 저장 파일명은 `<google-docs-id>-images-<timestamp>.zip`입니다.
- ZIP 안의 이미지는 문서 등장 순서대로 `001`, `002`, `003`처럼 번호가 붙습니다.
- 이미지는 재인코딩하지 않고 DOCX 내부 `word/media` 파일 바이트를 그대로 복사합니다.
- 2x2 이하의 DOCX placeholder 이미지는 제외합니다.
- ZIP은 Chrome 다운로드 목록으로 내려받습니다. 브리지는 임시 파일로만 만들고 다운로드 후 정리하므로 생성 JSON 저장 폴더에는 남기지 않습니다.

## 규칙 형식

기본 규칙은 기존처럼 2칸 배열입니다.

```json
["원문", "교정"]
```

문맥 예외가 필요한 경우에만 세 번째 옵션 객체를 붙일 수 있습니다.

```json
["의 수", " 수", { "rejectAfter": ["많"] }]
["끔하", "끔 하", { "rejectBefore": ["깔"] }]
```

현재 지원 옵션:

- `rejectBefore`: 매칭 직전 문자열이 있으면 제외
- `rejectAfter`: 매칭 직후 문자열이 있으면 제외

`rules.json` 정본은 Google Sheets 규칙 시트입니다. 시트를 수정한 뒤 루트에서 다음 명령을 실행합니다.

```bash
python3 tools/sync_rules.py
```

`tools/sync_rules.py`는 시트 C열의 옵션 JSON을 읽고, C열이 비어 있으면 기존 `rules.json`의 같은 룰 옵션을 보존합니다.

## 개발

테스트:

```bash
node test/engine.test.mjs
```

문법 검사:

```bash
node --check content/docs.js
node --check background.js
node --check popup.js
node --check options.js
node --check tools/toytype_ai_bridge.mjs
node --check tools/toytype_ai_bridge_ctl.mjs
python3 -m py_compile tools/sync_rules.py
```

Node 20 이상을 권장합니다.

## 제한사항

- Google Docs 본문 인라인 밑줄은 제공하지 않습니다.
- 일반 웹페이지에서는 서로 다른 DOM 요소 경계에 걸친 오탈자를 잡지 못합니다.
- 페이지 또는 문서당 표시 한도는 500건입니다.
- 규칙은 문맥 전체를 이해하지 않는 문자열 매칭이므로 거짓 양성이 있을 수 있습니다.
- URL, 코드, 식별자 내부 영문 규칙은 가능한 한 자동 제외합니다.
- 규칙 시트의 순차 치환 전용 규칙 일부는 정상 표기까지 과검출하므로 엔진에서 제외합니다.

## 개인정보

기본 내장 규칙 검사는 외부 서버로 문서 내용을 전송하지 않습니다.

AI 교정 생성, AI 발문 삽입, AI 문장 길이 조절은 사용자가 로컬 브리지를 실행하고 직접 요청했을 때만 동작합니다. AI 교정은 현재 Google Docs 본문, 내장 규칙 참고 자료, fact-check용 문맥을, 발문 삽입은 커서 주변 문맥을, 문장 길이 조절은 선택 원문과 앞뒤 문맥을 설정된 로컬 CLI(Codex 또는 Claude Code)에 전달합니다. fact-check 패스는 웹 검색/웹 페이지 조회 도구를 사용할 수 있습니다. 해당 CLI의 동작 방식에 따라 외부 AI 서비스로 전송될 수 있습니다.

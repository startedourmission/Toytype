# 오탈자 레이더

골든래빗 편집자용 크롬 확장 프로그램입니다. 지금 보고 있는 웹페이지와 구글 독스 원고에서 오탈자를 찾아 **표시**합니다. 규칙은 골든래빗 어문 교정 마스터 시트에서 가져온 8개 카테고리, 6,854개 교정 규칙(오기 → 교정어 쌍)입니다.

> **표시만 하고 절대 수정하지 않습니다.** (회사 규칙)
> 페이지든 구글 독스 문서든, 이 확장이 텍스트를 고치거나 문서에 흔적을 남기는 일은 없습니다. 교정은 항상 편집자가 직접 합니다.

## 설치

1. 크롬 주소창에 `chrome://extensions` 입력 후 이동합니다.
2. 우측 상단의 **개발자 모드**를 켭니다.
3. **압축해제된 확장 프로그램을 로드** 버튼을 누르고, 이 폴더(`typo-radar`)를 선택합니다.
4. 툴바에 퍼즐 모양 아이콘으로 등록되면 끝입니다. (별도 아이콘은 없습니다.)

내 컴퓨터에 저장된 HTML 파일(`file://`로 열리는 페이지)도 검사하려면, 확장 세부정보에서 **"파일 URL에 대한 액세스 허용"**을 켜세요.

## 사용법

### 툴바 배지

현재 탭에서 발견된 오탈자 개수가 확장 아이콘 위에 빨간 배지로 표시됩니다.

### 일반 웹페이지

오탈자 위치에 물결 밑줄과 옅은 배경색이 칠해집니다. 색은 카테고리 계열을 뜻합니다.

- 빨강 — 표기 변환·맞춤법·맨마지막 (의미 오류)
- 보라 — 존대와 복수·높임말 서술어 (문체)
- 주황 — 공백 관련 3종

### 팝업 (툴바 아이콘 클릭)

- 발견된 오탈자 목록을 앞뒤 문맥과 함께 보여줍니다. `오기 → 교정어` 형태로 표시되며, 공백이 규칙에 포함된 경우 `␣` 기호로 보입니다(예: `␣부터 → 부터`).
- **항목을 클릭하면 교정어가 클립보드에 복사**됩니다. 바로 붙여넣어 고치면 됩니다.
- 카테고리별 체크박스로 검사 범위를 켜고 끌 수 있습니다. 바꾸면 즉시 다시 검사하고, 설정은 다른 탭에도 유지됩니다.
- **[이 사이트에서 끄기]** 버튼으로 특정 사이트에서 검사를 통째로 끌 수 있습니다. [다시 켜기]로 복구합니다.
- **[다시 검사]** 버튼으로 수동 재검사할 수 있습니다.

### 구글 독스

문서 화면 우하단에 작은 원형 버블이 뜹니다(문서를 연 뒤 약 3초 후 자동 검사). 버블의 숫자가 발견 건수입니다.

- 버블을 클릭하면 패널이 펼쳐지고, 카테고리별 발견 목록(앞뒤 문맥, 줄 번호 `¶N`)이 보입니다.
- 항목을 클릭하면 교정어가 복사됩니다. 문서 안 위치로 자동 이동은 안 되니, 문맥 스니펫을 Ctrl+F(Mac: Cmd+F)로 찾아가세요.
- **[다시 검사]** 버튼으로 재검사합니다. 검사는 문서의 **저장된 스냅샷 기준**이라 방금 타이핑한 내용은 수 초 늦게 반영될 수 있습니다. 연속 재검사는 10초 간격으로 제한됩니다(10초 안에 누르면 직전 텍스트 기준으로 다시 계산).
- 구글 독스에서는 본문에 밑줄이 그어지지 않고 패널 목록으로만 표시됩니다.

규칙 정본은 골든래빗 어문 교정 마스터 시트(`1TONriTcFIHiT9Bstm0pv9iH0QiBjfBpodeo02CiY98Y`)입니다. 시트를 수정한 뒤:

1. 이 폴더에서 `python3 tools/sync_rules.py` 실행 → 루트의 `rules.json`이 재생성됩니다. (gws CLI 인증이 필요합니다. 자세한 사용법은 `tools/sync_rules.py` 상단 주석 참조.)
2. `chrome://extensions`에서 오탈자 레이더의 새로고침(↻) 버튼을 누릅니다.

## 제한사항

- **구글 독스**: 본문 인라인 밑줄 없음(패널 목록만). 저장된 스냅샷 기준이라 최신 타이핑이 몇 초 늦게 반영될 수 있습니다.
- **일반 페이지**: 서로 다른 요소 경계에 걸친 오탈자는 못 잡습니다. 화면에 안 보이는 숨은 텍스트가 카운트에 포함될 수 있습니다.
- 페이지(문서)당 표시 한도는 **500건**입니다. 초과분은 잘리고 경고가 표시됩니다.
- 규칙이 문맥을 보지 않는 글자 그대로의 치환 매칭이라 **거짓 양성이 있을 수 있습니다.** 의심스러우면 카테고리 토글로 범위를 좁혀서 보세요. (영문 단어 규칙은 `GitHub` 같은 합성어나 URL·코드 안에서는 자동으로 걸리지 않게 처리되어 있습니다.)
- 마스터 시트의 **순차 치환 전용 규칙 12개**(`것 → ␣것`, `변수 → ␣변수`처럼 일단 전부 띄운 뒤 별도 보정 규칙으로 합성어를 되돌리는 규칙)는 단독 검출이 불가능해(이것·그때·매개변수 같은 정상 표기까지 전부 발화) 검사에서 제외됩니다. 붙여쓰기 오류는 `는것`·`할때`·`콜백함수` 같은 명시적 규칙들이 잡습니다.

## 🟠 E. 골든 어문 규정 위반 / 과잉 분리

proofread.md 규정과 충돌하거나, 멀쩡한 단어를 잘못 쪼개 **오검출을 만드는** 규칙.

**(1) 보조용언/명사를 과하게 분리해 false positive 생성** — `해보`/`어보`류가 뒤따르는 명사(보통·보안·보조)를 잘못 끊음:

| 위치 | 규칙 | 문제 |
|---|---|---|
| space3#510, final#279 | `해보통`→`해 보통` | '보통'(명사)을 끊음 |
| final#286 | `있어보안`→`있어 보안` | '보안'을 끊음 |
| final#289 | `해보안`→`해 보안` | '보안'을 끊음 |
| final#205 | `위해보조`→`위해 보조` | '보조'를 끊음 |

> 이건 "해보다→해 보다" 식 분리 규칙이 명사 앞에서 오발한 흔적. 해당 규칙은 삭제하거나 `rejectAfter`로 보통/보안/보조 등을 막아야 한다.

**(2) 골든 예외 확인 필요** — 골든은 "**'있다'는 앞말과 띄어쓴다**"(서 있다, 들어 있다)가 **명시적 예외**다. 따라서 `되어 있`·`들어 있`·`남아 있`·`비어 있` 류 분리 규칙은 **올바름**(위반 아님). 다만 `많아 보입니다`·`고 싶`처럼 보조용언 `보다/싶다`는 골든 "보조용언 붙여쓴다" 원칙상 **붙여야** 하는데 현재 띄우고 있어, 원칙과 어긋난다 — 정책 확정 후 일괄 처리 권장.

**(3) em대시/중앙점 본문 금지** (골든 §6): rules.json 본문 교정엔 특수문자 치환이 거의 없으나, source 시트에 `—`·`·` 정리 규칙을 추가하면 골든 §6을 자동 점검할 수 있음(현재 미커버).
